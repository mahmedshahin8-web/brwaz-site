# تقرير هندسة مشروع "برواز" الموجه لنموذج (Qwen 3.5)

هذا التقرير مخصص ليكون المرجعية الهندسية المعمارية (Architecture Document) لنموذج Qwen 3.5 لكي يعيد بناء مشروع "برواز" بمستوى احترافي.

## 1. نظرة عامة على النظام (System Overview)
"برواز" هو محرك إنتاج محتوى (Content Production Studio) يعمل بالذكاء الاصطناعي، يهدف لتوليد حلقات وثائقية / يوتيوب كاملة (سكريبت، مشاهد، توجيهات بصرية، هندسة صوتية، ومقاطع قصيرة) بأسلوب "الدحيح" أو الصحافة الاستقصائية المصرية (العامية المصرية الراقية). النظام لا يكتب محتوى إنشائياً فقط، بل يقوم بالبحث في المصادر الحقيقية أولاً، ثم يبني هيكل درامي، ثم يكتب الإسكربت، ثم يوجه فريق الموشن جرافيك والمخرج الفني.

## 2. بنية المجلدات الأساسية (Directory Structure)
المشروع مبني بـ React (Vite + TypeScript) بالإضافة إلى Tailwind CSS للواجهات.

- `src/types.ts`: يحتوي على كافة الواجهات (Interfaces) والهياكل البيانية.
- `src/lib/gemini.ts`: "مخ النظام" والمسؤول عن الاتصال بـ API الذكاء الاصطناعي (يدعم Gemini محلياً عبر شبكة أو Ollama). يحتوي على الـ RAG، والـ Prompts المعقدة، والـ Nodes المتسلسلة.
- `src/lib/agents.ts`: يحتوي على تصنيفات الذكاء الاصطناعي (مثل Scriptwriter، Director، Art Director).
- `src/core/config/dna.ts`: يحتوي على 23 "مود" أو "قالب" صحفي لأسلوب الفيديوهات.
- `src/core/identity`: يحتوي على قواعد السرد لمنع نبرة "الروبوت" ومنع الفصحى الجامدة، وضمان استخدام اللهجة المصرية بأساليب مختلفة (مثلاً: طريقة الدحيح، المحقق، سرد القهاوي).
- `src/pages/ContentCreationPage.tsx`: واجهة المستخدم الرئيسية المكونة من (غرفة الأخبار -> تحديد المانشيت -> هيكل التحقيق -> الطبعة النهائية). تتعامل مع حالات الـ Loading المتقدمة وحركات السحب وحفظ الـ Dossier.
- `src/services/`: للتعامل مع الـ Text-to-Speech (مثل ElevenLabs) ولتوليد الصور وغيرها.

## 3. محرك الأنابيب التسلسلي (The Orchestrator Logic & Nodes)
يعتمد النظام على هندسة Nodes تسلسلية حتى لا يُهدر الـ Context Window ويخرج المحتوى بجودة ممتازة:

### 🌟 Node 0: OSINT & RAG Engine (البحث الجنائي والبحث مفتوح المصدر)
- **وظيفته:** تلقي الموضوع والـ Mood المطلوب، واستخدام (Google Search Grounding) للبحث عن **تواريخ حقيقية، أحداث، وشخصيات فعلية**.
- **المخرج (OsintDossier):** يطلع JSON يحتوي على: الملخص، التايم لاين، الكيانات، التناقضات المخفية (Hidden patterns)، والمصادر الحقيقية الموثقة مع رابطها، والـ Visual Anchors (تفاصيل بصرية تاريخية واقعية).

### 🌟 Node 1: Story Architect (مهندس القصة)
- **وظيفته:** يأخذ مخرجات الـ OSINT ويبني هيكل المشاهد (Chapters/Scenes). يحسب المدة الزمنية المطلوبة، ويولد لكل مشهد (Scene) "حقيقة جوهرية" (Core Fact) ووصف بصري مجرد.

### 🌟 Node 2: Master Scriptwriter & Director (الكاتب والمخرج)
- **وظيفته:** يأخذ الهيكل ويبدأ بكتابة التعليق الصوتي (`voiceover_text`) **بالعامية المصرية دارجة الصميمية**. يضع وقفات درامية `[صمت درامي]`. يكتب توجيهات المونتاج و الـ Cinematic Movement و English Image Prompt لميدجورني/رانواي.
- **القاعدة الذهبية (Identity Guard):** يُمنع تماماً وصف بشر أو وجوه صريحة في الـ Image Prompts (Faceless Rule) بحيث يتم بناء الفيديوهات بناءً على "الموتيفات" كالخرائط، الأوراق الصفراء، الدخان، الخ. ويُمنع استخدام كليشيهات الـ AI (عزيزي المشاهد، في الختام...الخ).

### 🌟 Node 3: Packaging & Shorts (التغليف والمقاطع القصيرة)
- **وظيفته:** يولد عنوان يوتيوب، وصف بأسلوب الدحيح، أفكار لـ Shorts (طعم الـ Hook، المحتوى، الـ CTA للتحويل للحلقة الطويلة).

## 4. المودات الـ 23 والمصادر التخيلية المقترنة بها (The 23 Moods)
يحدد النظام زاوية البحث (Research Angle) وعقار السرد وأسلوبه بناءً على المود، ويقترح للـ AI مصادر تاريخية يبحث فيها:
1. أرشيف الضلمة (تحقيقات المحاكم وجرائد الأهرام القديمة)
2. كلاكيت وتزوير (تحليل وهم الإعلام)
3. ملفات متقفلش (ألغاز وغموض)
4. خرافات شعبية (تفكيك أساطير مثل النداهة والسلعوة بحثياً)
5. حكاوي الأجداد (نوستالجيا الرعب)
6. سبوبة ولا ابتكار (قصص البيزنس الوهمي)
7. حواديت شوارع (تاريخ المقاهي والحارات)
8. صراع العروش العربي (التاريخ السياسي المكيافيلي)
9. تكنولوجيا مرعبة (السايبر سكيورتي والاختراقات)
10. اقتصاد الشارع (اقتصاد الظل والأسواق السوداء)
11. ملفات مخابراتية (تاريخ الجواسيس)
12. طريقة الدحيح (Pop-science وتبسيط العلوم بأمثلة شعبية الميكروباص)
13. خرائط دموية (شرح جيوسياسي بالخرائط)
14. سبورة بيضاء (شرح تعليمي)
15. ميمز ومقاطع (قضايا تريند بأسلوب ساخر)
16. رحلة في عقل مجرم (تشريح نفسي لجرائم حقيقية)
17. المستقبل الديسطوبي (سيناريوهات نهاية العالم)
18. محاكمة التاريخ (مرافعة محكمة حول شخصيات جدلية)
19. اقتصاد البقاء (كيف يعيش الناس وقت المجاعات أو التضخم)
20. جبل الجليد (الرحلة من السطح لأعمق النظريات وتنتشر مؤخرًا جدًا)
21. همس الحكايات (Dark ASMR الاعتماد على المؤثرات الصوتية)
22. شريط ملعون (توثيق فيديوهات مهجورة Found Footage)
23. مسافر عبر الزمن (شرح بديهياتنا الحالية كأنها خيال للماضي).

## 6. واجهة المستخدم وتجربة المستخدم (UI/UX)
- تعتمد على اللون العنابي (#8b0000)، والبيج الداكن/الورق العتيق (#f4eee0) والأسود لتبدو الواجهة كأنها "أرشيف استخباراتي قديم" أو مكتب صحيفة.
- الواجهة تقوم بعرض حالة التحميل برسائل متغيرة تعطي إحساس للمستخدم بأن النظام يقلب في صفحات الدفاتر والميكروفيلم (مثال: "[!] يتم الآن التنقيب في سجلات التحقيق...").
- يوجد تصدير للـ Audio (عن طريق ElevenLabs مقطعة حسب المشاهد)، ومستند السكريبت Text، وتصدير مدمج في ملف Zip كحزمة إنتاج Production Asset.

## توجيه مخصص لـ Qwen 3.5:
عند استلامك لهذه المعلومات في البرومبت القادم لبناء النظام، يجب التركيز على:
1. فصل الـ Prompts عن كود الـ UI.
2. تزويد المحرك الرئيسي بخاصية "Retry" مع الـ Fallback لتفادي أخطاء الـ API أو أخطاء الـ JSON parse.
3. إدارة الـ State داخلياً بكفاءة (يُفضل تنظيف المكون الطويل `ContentCreationPage.tsx` وتقسيمه برمجياً إلى مكونات أصغر: `DossierView` و `ScriptView` و `ExportStation`).
4. التركيز الشديد على محرك (Identity Guard) في السيرفر أو في اللوجيك لمنع الذكاء الاصطناعي من التحول إلى لغة روبوتية أو توليد صور تحتوي على أشخاص.
